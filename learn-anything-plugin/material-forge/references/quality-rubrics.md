# Material Quality Rubrics

Every material generated by the Material Forge must pass these checks before presentation. If any check fails, regenerate.

## Universal Four-Check Rubric

### Check 1: Accuracy
- All factual claims must be verifiable
- Math and logic must be verified step-by-step
- Code examples must be syntactically correct and produce the stated output
- Domain terminology must be used correctly
- If uncertain about a fact, flag it rather than state it confidently

### Check 2: Difficulty Calibration
- Material difficulty matches the target task class level
- Prerequisites are within the learner's demonstrated knowledge (check knowledge graph)
- New concepts introduced per item <= 3 (cognitive load limit)
- If the learner has transfer advantage for this topic, difficulty should be adjusted upward

### Check 3: Pedagogical Quality
- Builds on prior concepts (references or uses previously learned material)
- Includes at least one active engagement opportunity (question, exercise, prompt)
- Avoids unnecessary jargon (or defines jargon on first use)
- Uses concrete examples before abstract definitions
- Varied surface features across exercises (don't repeat the same scenario)

### Check 4: Red-Team Pass
- Could a learner misinterpret this?
- Does it inadvertently teach a misconception?
- Are there edge cases where the stated rule doesn't apply (that should be noted)?
- Could the answer be correct for a different reason than intended (testing the wrong thing)?

## Type-Specific Quality Criteria

### SRS Flashcards
- Follows Matuschak's five principles (focused, precise, consistent, tractable, effortful)
- No anti-patterns (kitchen sink, ambiguous cloze, yes/no, shopping list, pattern matching)
- Component_id is present and correct
- Topic tags are hierarchical and accurate
- Difficulty estimate is reasonable for the target learner

### Worked Examples with Fading
- First example is fully worked with clear step-by-step explanation
- Each step has a self-explanation prompt
- Backward fading: last step removed first, then progressively earlier steps
- Fading pace matches learner level (novice=slow, intermediate=standard, advanced=fast)
- Final problem in the sequence requires full independent performance

### Productive Failure Scenarios
- Problem activates prior knowledge (references something the learner already knows)
- Multiple valid solution approaches exist (not just one right way)
- Challenge level is appropriate (hard but not impossible)
- NO scaffolding, hints, or correction embedded in the problem statement
- Consolidation instruction is prepared (to deliver AFTER the struggle)
- Consolidation explicitly bridges learner's likely attempts to the canonical solution

### Interleaved Practice Sets
- 25% current topic, 75% review
- No two consecutive problems use the same strategy
- At least one discrimination pair (similar surface, different deep structure)
- Difficulty is calibrated to the target accuracy range (75-85%)

### Reference One-Pagers
- Fits on one page (or one screen) when formatted
- Contains rules/principles, not just information
- Organized for quick reference (not sequential reading)
- Includes 2-3 examples showing the rules in action
- No filler — every element serves a reference purpose

### Assessment Instruments
- Items match the specified Bloom's level
- Surface features differ from instruction (tests transfer, not memory)
- Scoring criteria are explicit and unambiguous
- Multiple items per knowledge component (minimum 3)
- Gaming-resistant (open-ended, explanation-requiring, varying surface features)

### Mermaid Dependency Graphs
- 20 or fewer nodes per diagram (split larger graphs into module views)
- Color-coded by mastery status (green/yellow/gray/dashed)
- Uses `flowchart TD` with subgraphs for module grouping
- Node labels are human-readable (not vertex IDs)
- Prerequisite arrows point from prerequisite to dependent
